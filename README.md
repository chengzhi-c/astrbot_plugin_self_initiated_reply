# 主动回复

白名单会话内的 AstrBot 主动接话插件。

当前版本：`0.6.2`

插件会在白名单会话收到普通聊天消息后，等待一段静默时间，再通过“判断模型”决定是否适合主动接一句。真正生成回复时不走裸模型调用，而是复用 AstrBot 主 Agent / tool loop 管线，因此 Stealer 表情包工具、LivingMemory 召回与写入钩子会像正常 `@Bot` 回复一样参与。

直接 `@Bot`、唤醒词和指令仍交给 AstrBot 主回复链处理，避免主动回复和正常回复同时触发。

## 工作方式

1. **触发收集**
   - 插件监听白名单会话中的普通聊天消息。
   - 明确要求 Bot 接话、发表情包、发图、找图、来个动图等内容会优先放行。
   - 普通聊天会进入判断模型。

2. **判断模型只负责是否接话**
   - 判断模型使用可配置提示词输出严格 JSON：
     ```json
     {"should_reply": true, "reason": "一句简短理由"}
     ```
   - 它不负责生成正文，也不负责调用工具。

3. **回复生成走 AstrBot 主 Agent**
   - 主动回复正文通过 AstrBot 主 Agent / tool loop 生成。
   - 因此 `search_emoji`、`send_emoji_by_id` 等 Stealer 工具可以被模型正常调用。
   - LivingMemory 的 `on_llm_request` / `on_llm_response` 钩子也会自然参与召回和写入。

4. **不直连其他插件数据库**
   - 本插件不再直接读取或写入 Stealer、LivingMemory 的数据库。
   - 跨插件能力由 AstrBot 主回复链、工具系统和事件钩子完成。

## 设置页面

插件详情页中的设置页面用于管理主动回复自身的触发和判断逻辑，主要包含：

- 插件运行状态和白名单数量
- 主动回复运行开关
- 最少上下文消息数、空闲触发秒数、冷却秒数
- 判断模型 Provider 选择/手动输入、温度、超时秒数
- 判断提示词、实时预览与恢复默认提示词按钮
- 可展开的提示词变量帮助
- 高级白名单 UMO 批量编辑

### 判断模型调参

页面中的“判断模型调参”区域是最常用的调试入口。

可调字段：

- **启用提示词判断模型**：关闭后普通消息不会由判断模型主动接话；明确接话/表情包请求仍可按规则放行。
- **判断模型 Provider**：默认使用当前会话模型；也可以从已有 LLM Provider 下拉选择，或切换到手动输入填写特殊 Provider ID。
- **判断温度**：建议 `0.0 - 0.3`，越低越稳定。
- **判断超时秒数**：判断模型最大等待时间，默认建议 `20` 秒。
- **判断提示词**：控制“什么时候该主动接话”。右侧预览会用示例变量实时渲染，方便检查最终提示词形态。

提示词可用变量：

```text
{session}                 # 当前会话 UMO，用于区分群聊/私聊/平台
{trigger}                 # 触发类型，例如 message、patrol、manual
{bot_aliases}             # Bot 名称/别名列表，便于判断用户是否在叫 Bot
{latest_message}          # 最近一条用户消息文本
{recent_messages}         # 最近上下文消息，通常是判断是否接话的主要依据
{last_message_age_sec}    # 距离最近一条消息过去的秒数
{last_reply_age_sec}      # 距离本插件上次主动回复过去的秒数
```

如果提示词留空保存，会恢复默认提示词。

预览窗口只使用页面内置的示例变量，不读取真实群聊消息；实际运行时，插件会根据当前会话、触发方式和最近消息替换这些变量。

修改配置后点击页面顶部或底部的“保存”。保存成功后页面会回读服务端配置；如果重载插件或重启 AstrBot 后仍要查看最新页面资源，建议在浏览器中硬刷新。

## 指令

为避免日常中文聊天误触发管理指令，插件只注册英文命令入口。

| 指令 | 说明 |
| --- | --- |
| `/selfreply` | 查看帮助 |
| `/selfreply status` | 查看运行状态、判断模型和白名单信息 |
| `/selfreply add` | 将当前会话加入主动回复白名单 |
| `/selfreply remove` | 将当前会话移出主动回复白名单 |
| `/selfreply list` | 查看白名单 |
| `/selfreply check [content]` | 手动测试一次主动回复，可附带测试内容 |
| `/selfreply on` | 临时启用主动回复 |
| `/selfreply off` | 临时暂停主动回复 |
| `/selfreply debug` | 查看当前会话、发送者和触发识别信息 |

也支持 `@Bot selfreply add` 这类管理写法；普通 `@Bot` 聊天不会触发主动回复插件，而是交给 AstrBot 正常回复链。

## 关于白名单

主动回复白名单和 AstrBot 全局会话白名单是两层不同限制：

- AstrBot 全局白名单会先于插件生效。
- 如果某个群没有通过 AstrBot 全局白名单，消息会在到达插件前被核心拦截。
- 本插件白名单只决定“是否允许主动回复”。

在群聊中使用 `/selfreply add` 会把整个群会话加入白名单，而不是只加入当前发言者。

页面里仍保留“高级：白名单会话 ID”区域，适合批量编辑。群聊可以填写完整 UMO，也可以直接填写 QQ 群号；日常更建议使用 `/selfreply add` 和 `/selfreply remove`。

## 建议配置

- **判断温度**：建议 `0.0 - 0.3`。
- **判断超时秒数**：建议 `10 - 30` 秒。
- **最少上下文消息数**：建议 `3 - 8`，默认可保持 `5`。
- **空闲触发秒数**：建议 `30 - 120` 秒，测试时可临时调低。
- **冷却秒数**：建议 `300 - 900` 秒，避免刷屏。
- **每日每会话最大回复次数**：建议 `5 - 12` 次，`0` 表示不限。
- **忽略发送者 ID**：只在确实要屏蔽某个用户时填写；管理员不会被自动屏蔽。

## 和 Stealer / LivingMemory 的关系

本插件不直接管理 Stealer 和 LivingMemory。

正确关系是：

- 主动回复插件负责判断“要不要接话”。
- AstrBot 主 Agent 负责生成回复和调用工具。
- Stealer 插件负责表情包搜索、选择和发送。
- LivingMemory 插件负责长期记忆召回和写入。

因此，如果要调表情包数据，请去 Stealer 插件页面；如果要调长期记忆，请去 LivingMemory 插件页面。本插件页面只管理主动回复自己的触发和判断逻辑。

## 排查建议

如果没有主动回复：

1. 确认当前会话已加入主动回复白名单。
2. 确认 AstrBot 全局白名单没有拦截该会话。
3. 查看 `/selfreply status`。
4. 查看日志中的 `should_reply` 和 `reason`。
5. 如果判断模型经常不放行，优先调整页面里的“判断提示词”和“判断温度”。

如果不能发表情包：

1. 确认 Stealer 插件启用且工具可用。
2. 确认主动回复已经进入主 Agent 生成阶段，而不是被判断模型拦截。
3. 在提示词中明确允许“发表情包/找图/动图请求应 should_reply=true”。
4. 用普通 `@Bot 来个表情包` 测试 Stealer 本身是否正常。

修改插件代码、页面或 metadata 后，建议重启 AstrBot 或重载插件让改动生效。
